Connecting Namecheap Domain To GitHub Pages

October 16, 2025

Introduction

Connecting your Namecheap domain to GitHub Pages is one of quickest ways to give your project custom web address instead of the default GitHub Pages URL. This guide provides step-by-step process and screenshots to:

  1. Link your domain and Enable HTTPs on GitHub
  2. Configure DNS records on Namecheap

Prerequisites

Before we dive in, make sure you have:

  • A static webpage hosted on GitHub Pages
  • Your domain being pointed to a DNS by Namecheap

GitHub Pages Custom Domain setup

Go to your GitHub repository settings, scroll down to the "GitHub Pages" section, and enter your custom domain name.

Understanding Chat Mode Locations

Custom chat modes in VS Code can be stored in different locations depending on your needs. Understanding where to place your prompt files is crucial for effective organization and team collaboration.

Locating the Prompts Directory

The prompts directory is where you'll store all your custom chat mode files. By default, VS Code looks for prompts in your user settings directory, but you can also create workspace-specific prompts for project-based customization.

Creating Your First Custom Chat Mode

Now that we have the basics covered, let's create your first custom chat mode. This process involves creating a prompt file with specific instructions that define how the AI should behave.

Step 1: Create the Prompt File

Navigate to your prompts directory and create a new file with a descriptive name. The file should have a .md extension and contain clear, specific instructions for the AI.

Step 2: Advanced Customization

Once you've created a basic custom chat mode, you can enhance it with advanced features like context awareness, specific coding patterns, and domain-specific knowledge.

Best Practices for Custom Chat Modes

To get the most out of your custom chat modes, follow these best practices:

1. Be Specific and Detailed

The more specific your instructions, the better the AI can assist you. Include details about coding standards, preferred patterns, and common scenarios.

2. Include Context Examples

Providing examples helps the AI understand exactly what you're looking for. Include code snippets, expected outputs, and edge cases.

3. Define Response Structure

Specify how you want responses formatted. This could include code comment styles, documentation formats, or explanation structures.

4. Version Control Your Prompts

Store your custom prompts in version control so your team can share and iterate on them together. This ensures consistency across your development team.

Practical Examples

Let's look at some practical examples of custom chat modes that can enhance your development workflow:

  • Framework-Specific Assistant: Create a chat mode tailored to React, Vue, Angular, or any other framework you use regularly.
  • Code Review Assistant: Design a mode that focuses on code quality, security concerns, and best practices.
  • Documentation Generator: Build a chat mode that specializes in creating comprehensive documentation for your code.
  • Testing Assistant: Develop a mode that helps write unit tests, integration tests, and suggests edge cases.

Conclusion

Custom chat modes are a powerful way to extend GitHub Copilot's capabilities and tailor it to your specific development needs. By following this guide, you can create specialized AI assistants that understand your project's context, follow your team's coding standards, and help you write better code faster.

Start experimenting with custom chat modes today and discover how they can transform your development workflow. As you become more comfortable with the basics, explore advanced techniques like combining multiple prompts, using variables, and integrating with your CI/CD pipeline.
